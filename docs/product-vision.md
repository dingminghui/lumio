# Lumio 产品定义

## 一句话定位

**面向内容创作者的 AI 内容生产画布**——在一块可视化画布上，将多个 AI 节点连接成生产流水线，最终输出可直接使用的 HTML、PDF、演示文稿等交付物。

---

## 核心使用场景

用户想制作一个 **AI 健身助手 App 落地页**：

```
[节点 A：标题文案]  ──┐
[节点 B：产品介绍]  ──┼──→  [节点 C：合成 HTML 落地页]
[节点 D：配图生成]  ──┘
```

1. 在画布上新增"文案"节点，与 AI 对话生成标题文案
2. 新增"产品介绍"节点，生成介绍段落
3. 新增"配图"节点，生成插图
4. 新增"HTML 合成"节点，连接上面三个节点
5. AI 自动读取上游节点内容，合成完整 HTML 页面

**产物**：一个可直接部署的 HTML 落地页。

---

## 与竞品的本质区别

| 工具             | 定位                           | 局限                   |
| ---------------- | ------------------------------ | ---------------------- |
| ChatGPT / Claude | 对话生成内容                   | 线性，无法组合多步产物 |
| Notion AI        | 文档内 AI 辅助                 | 只有文档，无流水线     |
| Dify             | 给开发者搭 AI 工作流           | 面向工程师，产物是 API |
| Miro / FigJam    | 可视化协作画布                 | 无 AI 内容生成         |
| **Lumio**        | **给创作者用的 AI 内容生产线** | —                      |

---

## 节点类型（角色分工）

### 源节点（独立生成内容）

| 类型     | 产出          |
| -------- | ------------- |
| 文案节点 | Markdown 文本 |
| 插图节点 | 图片 / SVG    |
| 数据节点 | 表格 / JSON   |
| 代码节点 | 可执行代码    |

### 合成节点（消费上游，输出交付物）

| 类型         | 产出           |
| ------------ | -------------- |
| HTML 合成    | 完整 HTML 页面 |
| 报告合成     | Markdown / PDF |
| 演示文稿合成 | HTML Slides    |

合成节点运行时，**自动读取所有连接的上游节点内容**，作为 AI 的输入上下文。

---

## 画布的意义

画布不是"花哨的文件夹"，而是**流水线的可视化编排**：

- **节点** = 一个 AI 生产步骤
- **连线（边）** = 数据流，定义依赖关系
- **空间布局** = 生产逻辑的直觉表达

这是画布形态相对于线性工具的真正优势。
